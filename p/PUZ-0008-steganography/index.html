1

Learn — secrecy vs. hiding

Cryptography hides a message’s meaning. Steganography hides its existence.

Every pixel in a colour image is three numbers — red, green, blue — each 0–255. The lowest bit of each number barely matters: nudging a red value from 154 to 155 is a change of one part in 256, invisible to the eye. So you can overwrite those lowest bits with the bits of a secret message, and the picture looks exactly the same. That’s LSB steganography — least-significant-bit hiding.

Why it changes the puzzle

A ciphertext announces itself — it’s obviously scrambled. A stego image doesn’t: it looks like an ordinary photo. So the attacker’s first job isn’t decoding, it’s detection — is there a message in here at all? The tell is in the lowest bits: in a normal image they’re close to random noise, but hidden data leaves a tell-tale pattern. Pull the lowest bit out of every pixel and look at it directly, and a hidden message lights up.

Best use today: steganography and cryptography stack beautifully — encrypt first, then hide the ciphertext. An eavesdropper has to both notice the message and break it. Spies, watermarkers, and CTF authors all lean on exactly this.

Analyze — make the lowest bits visible

The steganalyst’s trick: throw away the top 7 bits and amplify the last one.

This takes the picture above and rebuilds it using only each pixel’s lowest bit — black for 0, white for 1. In a normal photo that’s near-random fuzz; where a message hides, structure appears. Hide a message in Station 2, then amplify and watch the band of hidden data show up.

how does “extract” know where to look? (spoiler)

The message was written into the lowest bit of each red/green/blue value, in reading order, prefixed by a 32-bit length. Extract reads that length, then pulls exactly that many bytes back out of the lowest bits and decodes them as text.
